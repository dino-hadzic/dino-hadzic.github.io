Poglavlje 3
Sortiranje

Sortiranje je temeljni problem u dizajnu algoritama. Mnogi efikasni algoritmi koriste sortiranje kao podproceduru (subroutine) jer je podatke često lakše obraditi ako su elementi u sortiranom redoslijedu.

Na primjer, problem "sadrži li polje dva jednaka elementa?" lako se rješava sortiranjem. Ako polje sadrži dva jednaka elementa, oni će nakon sortiranja biti jedan pokraj drugoga, pa ih je lako pronaći. Također, problem "koji je najučestaliji element u polju?" može se riješiti na sličan način.

Postoji mnogo algoritama za sortiranje i oni su također dobri primjeri kako primijeniti različite tehnike dizajna algoritama. Efikasni opći algoritmi za sortiranje rade u vremenu $O(n \log n)$, a mnogi algoritmi koji koriste sortiranje kao podproceduru također imaju ovu vremensku složenost.

Teorija sortiranja

Osnovni problem u sortiranju glasi kako slijedi:
Zadano je polje koje sadrži $n$ elemenata. Vaš zadatak je sortirati elemente u rastućem redoslijedu.

Na primjer, polje:
1, 3, 8, 2, 9, 2, 5, 6
nakon sortiranja izgledat će ovako:
1, 2, 2, 3, 5, 6, 8, 9

Algoritmi složenosti $O(n^{2})$

Jednostavni algoritmi za sortiranje polja rade u vremenu $O(n^{2})$. Takvi algoritmi su kratki i obično se sastoje od dvije ugniježđene petlje. Poznati algoritam za sortiranje vremenske složenosti $O(n^{2})$ je bubble sort (balončićasto sortiranje), gdje elementi "burbuljaju" (bubble) u polju prema svojim vrijednostima.

Bubble sort se sastoji od $n$ krugova. U svakom krugu algoritam prolazi kroz elemente polja. Kad god se pronađu dva uzastopna elementa koji nisu u ispravnom redoslijedu, algoritam ih zamjenjuje. Algoritam se može implementirati na sljedeći način:

for (int i = 0; i < n; i++) {
    for (int j = 0; j < n - 1; j++) {
        if (array[j] > array[j + 1]) {
            swap(array[j], array[j + 1]);
        }
    }
}

Nakon prvog kruga algoritma, najveći element bit će na ispravnom mjestu, a općenito, nakon $k$ krugova, $k$ najvećih elemenata bit će na ispravnim pozicijama. Dakle, nakon $n$ krugova, cijelo će polje biti sortirano.

Inverzije

Bubble sort je primjer algoritma za sortiranje koji uvijek zamjenjuje uzastopne elemente u polju. Ispostavlja se da je vremenska složenost takvog algoritma uvijek najmanje $O(n^{2})$, jer je u najgorem slučaju potrebno $O(n^{2})$ zamjena za sortiranje polja.

Koristan koncept pri analizi algoritama za sortiranje je inverzija: par indeksa polja $(array[a], array[b])$ takav da je $a < b$ i $array[a] > array[b]$, odnosno elementi su u pogrešnom redoslijedu. Broj inverzija pokazuje koliko je posla potrebno da bi se polje sortiralo. Polje je potpuno sortirano kada nema inverzija. S druge strane, ako su elementi polja u obrnutom redoslijedu, broj inverzija je najveći mogući:

$$1+2+\cdot\cdot\cdot+(n-1)=\frac{n(n-1)}{2}=O(n^{2})$$

Zamjena para uzastopnih elemenata koji su u pogrešnom redoslijedu uklanja točno jednu inverziju iz polja. Stoga, ako algoritam za sortiranje može zamijeniti samo uzastopne elemente, svaka zamjena uklanja najviše jednu inverziju, a vremenska složenost algoritma je najmanje $O(n^{2})$.

Algoritmi složenosti $O(n \log n)$

Moguće je efikasno sortirati polje u vremenu $O(n \log n)$ koristeći algoritme koji nisu ograničeni na zamjenu uzastopnih elemenata. Jedan takav algoritam je merge sort (sortiranje spajanjem), koji se temelji na rekurziji.

Merge sort sortira podpolje array[a...b] na sljedeći način:

  1. Ako je $a = b$, ne čini ništa jer je podpolje već sortirano.
  2. Izračunaj poziciju srednjeg elementa: $k = \lfloor(a+b)/2\rfloor$.
  3. Rekurzivno sortiraj podpolje array[a...k].
  4. Rekurzivno sortiraj podpolje array[k+1...b].
  5. Spoji (merge) sortirana podpolja array[a...k] i array[k+1...b] u sortirano podpolje array[a...b].

Merge sort je efikasan algoritam jer u svakom koraku prepolovi veličinu podpolja. Rekurzija se sastoji od $O(\log n)$ razina, a obrada svake razine traje $O(n)$ vremena. Spajanje podpolja moguće je u linearnom vremenu jer su ona već sortirana.

Donja granica sortiranja

Je li moguće sortirati polje brže od vremena $O(n \log n)$? Ispostavlja se da to nije moguće kada se ograničimo na algoritme za sortiranje koji se temelje na uspoređivanju elemenata polja. Donja granica vremenske složenosti može se dokazati promatranjem sortiranja kao procesa u kojem svaka usporedba dvaju elemenata daje više informacija o sadržaju polja. Rezultati tog procesa daju ukupno $n!$ načina za sortiranje polja. Iz tog razloga, visina stabla odlučivanja mora biti najmanje:

$$\log_{2}(n!)=\log_{2}(1)+\log_{2}(2)+\cdot\cdot\cdot+\log_{2}(n)$$

Donju granicu za ovu sumu dobivamo odabirom posljednjih $n/2$ elemenata i promjenom vrijednosti svakog elementa u $\log_{2}(n/2)$. To daje procjenu:

$$\log_{2}(n!) \ge (n/2) \cdot \log_{2}(n/2)$$

pa je visina stabla i minimalni mogući broj koraka u algoritmu sortiranja u najgorem slučaju najmanje $n \log n$.

Counting sort

Donja granica $n \log n$ ne vrijedi za algoritme koji ne uspoređuju elemente polja, već koriste neke druge informacije. Primjer takvog algoritma je counting sort (sortiranje prebrojavanjem) koji sortira polje u vremenu $O(n)$ pod pretpostavkom da je svaki element u polju cijeli broj između $0 \dots c$ i da je $c = O(n)$.

Algoritam stvara pomoćno polje za evidenciju (bookkeeping array), čiji su indeksi zapravo elementi originalnog polja. Algoritam prolazi kroz originalno polje i računa koliko se puta svaki element pojavljuje. Izgradnja ovog polja traje $O(n)$ vremena, nakon čega se sortirano polje može kreirati u $O(n)$ vremena. Dakle, ukupna vremenska složenost counting sorta je $O(n)$. To je vrlo efikasan algoritam, ali se može koristiti samo kada je konstanta $c$ dovoljno mala.

Sortiranje u C++

Gotovo nikada nije dobra ideja koristiti vlastiti algoritam za sortiranje na natjecanju, jer su u programskim jezicima već dostupne izvrsne implementacije. Na primjer, standardna C++ biblioteka sadrži funkciju sort koja se lako može koristiti za sortiranje polja i drugih struktura podataka.

Sljedeći kod sortira vektor u rastućem redoslijedu:

vector<int> v = {4, 2, 5, 3, 5, 8, 3};
sort(v.begin(), v.end());

Zadani redoslijed sortiranja je rastući, ali obrnuti redoslijed je moguć na sljedeći način:

sort(v.rbegin(), v.rend());

Obično polje može se sortirati ovako:

int n = 7; // veličina polja
int a[] = {4, 2, 5, 3, 5, 8, 3};
sort(a, a + n);

Sljedeći kod sortira string s (što znači da se sortiraju njegovi znakovi):

string s = "monkey";
sort(s.begin(), s.end());

Operatori usporedbe

Funkcija sort zahtijeva da operator usporedbe bude definiran za tip podataka elemenata koji se sortiraju. Većina C++ tipova podataka ima ugrađen operator usporedbe. Parovi (pair) se sortiraju prvenstveno prema svom prvom elementu (first). Međutim, ako su prvi elementi dvaju parova jednaki, sortiraju se prema drugom elementu (second):

vector<pair<int, int>> v;
v.push_back({1, 5});
v.push_back({2, 3});
v.push_back({1, 2});
sort(v.begin(), v.end());

Strukture definirane od strane korisnika

Korisnički definirane strukture nemaju automatski operator usporedbe. Operator bi trebao biti definiran unutar strukture kao funkcija operator<:

struct P {
    int x, y;
    bool operator<(const P &p) {
        if (x != p.x) return x < p.x;
        else return y < p.y;
    }
};

Binarna pretraga

Opća metoda za traženje elementa u polju je korištenje petlje koja prolazi kroz elemente polja, što daje vremensku složenost $O(n)$. Međutim, ako je polje sortirano, pretragu možemo izvršiti puno brže jer nas redoslijed elemenata vodi kroz pretragu. Algoritam binarne pretrage efikasno pretražuje element u sortiranom polju u vremenu $O(\log n)$.

Metoda 1

Uobičajeni način implementacije binarne pretrage održava aktivnu regiju u polju, koja u početku sadrži sve elemente polja. U svakom koraku, pretraga provjerava srednji element aktivne regije. Ako je to ciljani element, pretraga završava. Inače se nastavlja u lijevoj ili desnoj polovici:

int a = 0, b = n - 1;
while (a <= b) {
    int k = (a + b) / 2;
    if (array[k] == x) {
        // x pronađen na indeksu k
    }
    if (array[k] > x) b = k - 1;
    else a = k + 1;
}

Metoda 2

Alternativna metoda temelji se na skokovima. Ideja je raditi skokove i usporavati kada se približimo ciljanom elementu. Početna duljina skoka je $n/2$, a u svakom koraku se polovi:

int k = 0;
for (int b = n / 2; b >= 1; b /= 2) {
    while (k + b < n && array[k + b] <= x) k += b;
}
if (array[k] == x) {
    // x pronađen na indeksu k
}

C++ funkcije

Standardna C++ biblioteka sadrži sljedeće funkcije temeljene na binarnoj pretrazi koje rade u logaritamskom vremenu:

  • lower_bound vraća pokazivač na prvi element polja čija je vrijednost najmanje $x$.
  • upper_bound vraća pokazivač na prvi element polja čija je vrijednost veća od $x$.
  • equal_range vraća oba navedena pokazivača.

Na primjer, sljedeći kod provjerava sadrži li polje element s vrijednošću $x$:

auto k = lower_bound(array, array + n, x) - array;
if (k < n && array[k] == x) {
    // x pronađen na indeksu k
}
