Poglavlje 4: Strukture podataka

Struktura podataka je način pohrane podataka u memoriji računala. Važno je odabrati odgovarajuću strukturu podataka za pojedini problem, jer svaka struktura ima svoje prednosti i nedostatke. Ključno pitanje glasi: koje su operacije učinkovite u odabranoj strukturi podataka?

Ovo poglavlje uvodi najvažnije strukture podataka u standardnoj biblioteci jezika C++. Preporučljivo je koristiti standardnu biblioteku kad god je to moguće jer će to uštedjeti mnogo vremena. Kasnije u knjizi naučit ćemo o naprednijim strukturama podataka koje nisu dostupne u standardnoj biblioteci.

Dinamički nizovi

Dinamički niz je niz čija se veličina može mijenjati tijekom izvođenja programa. Najpopularniji dinamički niz u jeziku C++ je struktura vector, koja se može koristiti gotovo isto kao i običan niz.

Sljedeći kod stvara prazan vektor i dodaje mu tri elementa:

vector<int> v;
v.push_back(3); // [3]
v.push_back(2); // [3,2]
v.push_back(5); // [3,2,5]

Nakon toga, elementima se može pristupiti kao u običnom nizu:

cout << v[0] << "\n"; // 3
cout << v[1] << "\n"; // 2
cout << v[2] << "\n"; // 5

Funkcija size vraća broj elemenata u vektoru. Sljedeći kod prolazi kroz vektor i ispisuje sve njegove elemente:

for (int i = 0; i < v.size(); i++) {
    cout << v[i] << "\n";
}
35

Kraći način za prolazak kroz vektor je sljedeći:

for (auto x : v) {
    cout << x << "\n";
}

Funkcija back vraća posljednji element u vektoru, a funkcija pop_back uklanja posljednji element:

vector<int> v;
v.push_back(5);
v.push_back(2);
cout << v.back() << "\n"; // 2
v.pop_back();
cout << v.back() << "\n"; // 5

Sljedeći kod stvara vektor s pet elemenata:

vector<int> v = {2, 4, 2, 5, 1};

Drugi način za stvaranje vektora je zadavanje broja elemenata i početne vrijednosti za svaki element:

// veličina 10, početna vrijednost 0
vector<int> v(10);

// veličina 10, početna vrijednost 5
vector<int> v(10, 5);

Unutarnja implementacija vektora koristi običan niz. Ako se veličina vektora poveća, a niz postane premalen, alocira se novi niz i svi se elementi premještaju u njega. Međutim, to se ne događa često pa je prosječna vremenska složenost operacije push_back jednaka $O(1)$.

Struktura string također je dinamički niz koji se može koristiti gotovo isto kao i vektor. Uz to, postoji posebna sintaksa za stringove koja nije dostupna u drugim strukturama podataka. Stringovi se mogu spajati pomoću simbola +. Funkcija substr(k, x) vraća podstring koji počinje na poziciji $k$ i ima duljinu $x$, a funkcija find(t) pronalazi poziciju prvog pojavljivanja podstringa $t$.

Sljedeći kod prikazuje neke operacije sa stringovima:

string a = "hatti";
string b = a + a;
cout << b << "\n"; // hattihatti
b[5] = 'v';
cout << b << "\n"; // hattivatti
string c = b.substr(3, 4);
cout << c << "\n"; // tiva
36

Strukture skupova (Set)

Skup (set) je struktura podataka koja održava kolekciju elemenata. Osnovne operacije nad skupovima su umetanje elemenata, pretraživanje i uklanjanje. Standardna C++ biblioteka sadrži dvije implementacije skupa: struktura set temelji se na balansiranom binarnom stablu i njezine operacije rade u vremenu $O(\log n)$. Struktura unordered_set koristi raspršivanje (hashing), a njezine operacije u prosjeku rade u vremenu $O(1)$.

Izbor implementacije skupa često je stvar ukusa. Prednost strukture set je u tome što održava poredak elemenata i pruža funkcije koje nisu dostupne u unordered_set. S druge strane, unordered_set može biti učinkovitiji.

Sljedeći kod stvara skup koji sadrži cijele brojeve i prikazuje neke od operacija. Funkcija insert dodaje element u skup, funkcija count vraća broj pojavljivanja elementa u skupu, a funkcija erase uklanja element iz skupa.

set<int> s;
s.insert(3);
s.insert(2);
s.insert(5);
cout << s.count(3) << "\n"; // 1
cout << s.count(4) << "\n"; // 0
s.erase(3);
s.insert(4);
cout << s.count(3) << "\n"; // 0
cout << s.count(4) << "\n"; // 1

Skup se uglavnom može koristiti poput vektora, ali elementima nije moguće pristupiti pomoću uglatih zagrada []. Sljedeći kod stvara skup, ispisuje broj elemenata u njemu, a zatim prolazi kroz sve elemente:

set<int> s = {2, 5, 6, 8};
cout << s.size() << "\n"; // 4
for (auto x : s) {
    cout << x << "\n";
}

Važno svojstvo skupova je da su svi njihovi elementi jedinstveni. Stoga funkcija count uvijek vraća ili 0 (element nije u skupu) ili 1 (element je u skupu), a funkcija insert nikada ne dodaje element u skup ako se on već tamo nalazi.

Sljedeći kod to ilustrira:

set<int> s;
s.insert(5);
s.insert(5);
s.insert(5);
cout << s.count(5) << "\n"; // 1
37

C++ također sadrži strukture multiset i unordered_multiset koje inače rade kao set i unordered_set, ali mogu sadržavati više instanci istog elementa. Na primjer, u sljedećem kodu sva tri pojavljivanja broja 5 dodaju se u multiset:

multiset<int> s;
s.insert(5);
s.insert(5);
s.insert(5);
cout << s.count(5) << "\n"; // 3

Funkcija erase uklanja sve instance elementa iz multiseta:

s.erase(5);
cout << s.count(5) << "\n"; // 0

Često želimo ukloniti samo jednu instancu, što se može učiniti na sljedeći način:

s.erase(s.find(5));
cout << s.count(5) << "\n"; // 2

Strukture mapa (Map)

Mapa je poopćeni niz koji se sastoji od parova ključ-vrijednost. Dok su ključevi u običnom nizu uvijek uzastopni cijeli brojevi $0, 1, \dots, n-1$ gdje je $n$ veličina niza, ključevi u mapi mogu biti bilo kojeg tipa podataka i ne moraju biti uzastopne vrijednosti.

Standardna C++ biblioteka sadrži dvije implementacije mapa koje odgovaraju implementacijama skupova: struktura map temelji se na balansiranom binarnom stablu i pristup elementima traje $O(\log n)$ vremena, dok struktura unordered_map koristi raspršivanje i pristup elementima u prosjeku traje $O(1)$ vremena.

Sljedeći kod stvara mapu u kojoj su ključevi stringovi, a vrijednosti cijeli brojevi:

map<string, int> m;
m["monkey"] = 4;
m["banana"] = 3;
m["harpsichord"] = 9;
cout << m["banana"] << "\n"; // 3

Ako se zatraži vrijednost ključa koji mapa ne sadrži, taj se ključ automatski dodaje u mapu s podrazumijevanom vrijednošću. Na primjer, u sljedećem kodu u mapu se dodaje ključ "aybabtu" s vrijednošću 0.

map<string, int> m;
cout << m["aybabtu"] << "\n"; // 0
38

Funkcija count provjerava postoji li ključ u mapi:

if (m.count("aybabtu")) {
    // ključ postoji
}

Sljedeći kod ispisuje sve ključeve i vrijednosti u mapi:

for (auto x : m) {
    cout << x.first << " " << x.second << "\n";
}

Iteratori i rasponi

Mnoge funkcije u standardnoj C++ biblioteci rade s iteratorima. Iterator je varijabla koja pokazuje na element u strukturi podataka.

Često korišteni iteratori begin i end definiraju raspon koji sadrži sve elemente u strukturi podataka. Iterator begin pokazuje na prvi element u strukturi podataka, a iterator end pokazuje na poziciju iza posljednjeg elementa. Situacija izgleda ovako:

$$\{3, 4, 6, 8, 12, 13, 14, 17\}$$

$$\uparrow \text{s.begin()} \qquad \dots \qquad \uparrow \text{s.end() (izvan strukture)}$$

Uočite asimetriju u iteratorima: s.begin() pokazuje na element unutar strukture podataka, dok s.end() pokazuje izvan nje. Stoga je raspon definiran iteratorima poluotvoren.

Rad s rasponima

Iteratori se koriste u funkcijama standardne C++ biblioteke kojima se zadaje raspon elemenata u strukturi podataka. Obično želimo obraditi sve elemente u strukturi, pa se funkciji prosljeđuju iteratori begin i end.

Na primjer, sljedeći kod sortira vektor pomoću funkcije sort, zatim okreće redoslijed elemenata pomoću funkcije reverse, te na kraju nasumično ispremiješa redoslijed elemenata pomoću funkcije random_shuffle.

sort(v.begin(), v.end());
reverse(v.begin(), v.end());
random_shuffle(v.begin(), v.end());

Ove se funkcije također mogu koristiti s običnim nizom. U tom slučaju, funkcijama se umjesto iteratora prosljeđuju pokazivači na niz:

39
sort(a, a + n);
reverse(a, a + n);
random_shuffle(a, a + n);

Iteratori skupova

Iteratori se često koriste za pristup elementima skupa. Sljedeći kod stvara iterator it koji pokazuje na najmanji element u skupu:

set<int>::iterator it = s.begin();

Kraći način za pisanje ovog koda je:

auto it = s.begin();

Elementu na koji iterator pokazuje može se pristupiti pomoću simbola *. Na primjer, sljedeći kod ispisuje prvi element u skupu:

auto it = s.begin();
cout << *it << "\n";

Iteratori se mogu pomicati pomoću operatora ++ (naprijed) i -- (natrag), što znači da se iterator pomiče na sljedeći ili prethodni element u skupu.

Sljedeći kod ispisuje sve elemente u uzlaznom redoslijedu:

for (auto it = s.begin(); it != s.end(); it++) {
    cout << *it << "\n";
}

Sljedeći kod ispisuje najveći element u skupu:

auto it = s.end(); 
it--;
cout << *it << "\n";

Funkcija find(x) vraća iterator koji pokazuje na element čija je vrijednost $x$. Međutim, ako skup ne sadrži $x$, iterator će biti jednak end.

auto it = s.find(x);
if (it == s.end()) {
    // x nije pronađen
}

Funkcija lower_bound(x) vraća iterator na najmanji element u skupu čija je vrijednost barem $x$, a funkcija upper_bound(x) vraća iterator na najmanji element u skupu čija je vrijednost veća od $x$. U obje funkcije, ako takav element ne postoji, vraćena vrijednost je end. Ove funkcije nisu podržane u strukturi unordered_set jer ona ne održava poredak elemenata.

40

Na primjer, sljedeći kod pronalazi element najbliži vrijednosti $x$:

auto it = s.lower_bound(x);
if (it == s.begin()) {
    cout << *it << "\n";
} else if (it == s.end()) {
    it--;
    cout << *it << "\n";
} else {
    int a = *it; 
    it--;
    int b = *it;
    if (x - b < a - x) cout << b << "\n";
    else cout << a << "\n";
}

Kod pretpostavlja da skup nije prazan i prolazi kroz sve moguće slučajeve pomoću iteratora it. Prvo, iterator pokazuje na najmanji element čija je vrijednost barem $x$. Ako je jednak begin, odgovarajući element je najbliži $x$. Ako je jednak end, najveći element u skupu je najbliži $x$. Ako nijedan od prethodnih slučajeva ne vrijedi, element najbliži $x$ je ili trenutni element na koji pokazuje it ili prethodni element.

Ostale strukture

Bitset

Bitset je niz čija je svaka vrijednost ili 0 ili 1. Na primjer, sljedeći kod stvara bitset koji sadrži 10 elemenata:

bitset<10> s;
s[1] = 1;
s[3] = 1;
s[4] = 1;
s[7] = 1;
cout << s[4] << "\n"; // 1
cout << s[5] << "\n"; // 0

Prednost korištenja bitseta je u tome što troše manje memorije od običnih nizova, jer svaki element u bitsetu koristi samo jedan bit memorije. Na primjer, ako se $n$ bitova pohrani u običan int niz, potrošit će se $32n$ bita memorije, dok odgovarajući bitset zahtijeva samo $n$ bita memorije. Osim toga, vrijednosti bitseta mogu se učinkovito manipulirati pomoću bitovnih operatora, što omogućuje optimizaciju algoritama.

Sljedeći kod prikazuje drugi način stvaranja gore navedenog bitseta (čita se s desna nalijevo):

bitset<10> s(string("0010011010")); // s desna nalijevo
cout << s[4] << "\n"; // 1
cout << s[5] << "\n"; // 0
41

Funkcija count vraća broj jedinica u bitsetu:

bitset<10> s(string("0010011010"));
cout << s.count() << "\n"; // 4

Sljedeći kod prikazuje primjere korištenja bitovnih operacija:

bitset<10> a(string("0010110110"));
bitset<10> b(string("1011011000"));
cout << (a & b) << "\n"; // 0010010000
cout << (a | b) << "\n"; // 1011111110
cout << (a ^ b) << "\n"; // 1001101110

Deque

Deque (dvostrani red) je dinamički niz čija se veličina može učinkovito mijenjati na oba kraja niza. Poput vektora, deque pruža funkcije push_back i pop_back, ali također uključuje funkcije push_front i pop_front koje nisu dostupne u vektoru.

Deque se može koristiti na sljedeći način:

deque<int> d;
d.push_back(5);  // [5]
d.push_back(2);  // [5,2]
d.push_front(3); // [3,5,2]
d.pop_back();    // [3,5]
d.pop_front();   // [5]

Unutarnja implementacija deque strukture složenija je od implementacije vektora i zbog toga je deque sporiji od vektora. Unatoč tome, i dodavanje i uklanjanje elemenata u prosjeku traju $O(1)$ vremena na oba kraja.

Stog (Stack)

Stog je struktura podataka koja pruža dvije operacije s vremenskom složenošću $O(1)$: dodavanje elementa na vrh i uklanjanje elementa s vrha. Moguće je pristupiti isključivo elementu koji se nalazi na samom vrhu stoga.

stack<int> s;
s.push(3);
s.push(2);
s.push(5);
cout << s.top(); // 5
s.pop();
cout << s.top(); // 2
42

Red (Queue)

Red također pruža dvije operacije s vremenskom složenošću $O(1)$: dodavanje elementa na kraj reda i uklanjanje prvog elementa iz reda. Moguće je pristupiti samo prvom i posljednjem elementu reda.

queue<int> q;
q.push(3);
q.push(2);
q.push(5);
cout << q.front(); // 3
q.pop();
cout << q.front(); // 2

Prioritetni red (Priority queue)

Prioritetni red održava skup elemenata. Podržane operacije su umetanje te, ovisno o vrsti reda, dohvaćanje i uklanjanje minimalnog ili maksimalnog elementa. Umetanje i uklanjanje traju $O(\log n)$ vremena, dok dohvaćanje traje $O(1)$ vremena.

Iako sortirani skup (set) učinkovito podržava sve operacije prioritetnog reda, prednost korištenja prioritetnog reda je u tome što ima manje konstantne faktore. Prioritetni red se obično implementira pomoću strukture gomile (heap) koja je mnogo jednostavnija od balansiranog binarnog stabla koje se koristi u sortiranom skupu.

Prema zadanim postavkama, elementi u C++ prioritetnom redu sortirani su u padajućem redoslijedu, što znači da pronalazimo i uklanjamo najveći element u redu. Sljedeći kod to ilustrira:

priority_queue<int> q;
q.push(3);
q.push(5);
q.push(7);
q.push(2);
cout << q.top() << "\n"; // 7
q.pop();
cout << q.top() << "\n"; // 5
q.pop();
q.push(6);
cout << q.top() << "\n"; // 6
q.pop();

Ako želimo stvoriti prioritetni red koji podržava pronalaženje i uklanjanje najmanjeg elementa, to možemo učiniti na sljedeći način:

priority_queue<int, vector<int>, greater<int>> q;

Strukture podataka temeljene na politikama (Policy-based)

Kompajler g++ također podržava neke strukture podataka koje nisu dio standardne C++ biblioteke. Takve strukture nazivaju se strukturama podataka temeljenim na politikama (policy-based data structures). Kako bismo koristili ove strukture, u kod moramo dodati sljedeće linije:

#include <ext/pb_ds/assoc_container.hpp>
using namespace __gnu_pbds;

Nakon toga možemo definirati strukturu podataka indexed_set koja je slična strukturi set, ali joj se može pristupati pomoću indeksa kao kod običnog niza. Definicija za cjelobrojne vrijednosti (int) glasi:

typedef tree<int, null_type, less<int>, rb_tree_tag,
             tree_order_statistics_node_update> indexed_set;

Sada možemo stvoriti skup na sljedeći način:

indexed_set s;
s.insert(2);
s.insert(3);
s.insert(7);
s.insert(9);

Posebnost ovog skupa je u tome što imamo pristup indeksima koje bi elementi imali u sortiranom nizu. Funkcija find_by_order vraća iterator na element na zadanoj poziciji:

auto x = s.find_by_order(2);
cout << *x << "\n"; // 7

A funkcija order_of_key vraća poziciju zadanog elementa:

cout << s.order_of_key(7) << "\n"; // 2

Ako se element ne pojavljuje u skupu, dobivamo poziciju koju bi taj element imao da se umetne u skup:

cout << s.order_of_key(6) << "\n"; // 2
cout << s.order_of_key(8) << "\n"; // 3

Obje funkcije rade u logaritamskom vremenu.

Usporedba sa sortiranjem

Često je moguće riješiti problem koristeći ili strukture podataka ili sortiranje. Ponekad postoje značajne razlike u stvarnoj učinkovitosti tih pristupa, što može biti skriveno u njihovim vremenskim složenostima.

Razmotrimo problem u kojem su nam zadane dvije liste $A$ i $B$ od kojih svaka sadrži $n$ elemenata. Naš je zadatak izračunati broj elemenata koji pripadaju objema listama. Na primjer, za liste $A = [5, 2, 8, 9, 4]$ i $B = [3, 2, 9, 5]$, odgovor je 3 jer brojevi 2, 5 i 9 pripadaju objema listama.

Izravno rješenje problema je prolazak kroz sve parove elemenata u vremenu $O(n^2)$, ali u nastavku ćemo se usredotočiti na učinkovitije algoritme.

Algoritam 1: Konstruiramo skup elemenata koji se pojavljuju u listi $A$, a nakon toga prolazimo kroz elemente liste $B$ i za svaki element provjeravamo pripada li također listi $A$. Ovo je učinkovito jer su elementi liste $A$ smješteni u skupu. Koristeći strukturu set, vremenska složenost ovog algoritma je $O(n \log n)$.

Algoritam 2: Nije nužno održavati sortirani skup, pa umjesto strukture set možemo koristiti strukturu unordered_set. Ovo je jednostavan način da algoritam učinimo učinkovitijim jer moramo promijeniti samo osnovnu strukturu podataka. Vremenska složenost novog algoritma je $O(n)$.

Algoritam 3: Umjesto struktura podataka, možemo koristiti sortiranje. Prvo sortiramo obje liste $A$ i $B$. Nakon toga istovremeno prolazimo kroz obje liste i pronalazimo zajedničke elemente. Vremenska složenost sortiranja je $O(n \log n)$, a ostatak algoritma radi u linearnom vremenu $O(n)$, pa je ukupna vremenska složenost $O(n \log n)$.

Usporedba učinkovitosti

Sljedeća tablica prikazuje koliko su navedeni algoritmi učinkoviti kada se veličina $n$ mijenja, a elementi lista su nasumični cijeli brojevi između $1$ i $10^9$:

$n$ Algoritam 1 (set) Algoritam 2 (unordered_set) Algoritam 3 (Sortiranje)
$10^6$ 1.5 s 0.3 s 0.2 s
$2 \cdot 10^6$ 3.7 s 0.8 s 0.3 s
$3 \cdot 10^6$ 5.7 s 1.3 s 0.5 s
$4 \cdot 10^6$ 7.7 s 1.7 s 0.7 s
$5 \cdot 10^6$ 10.0 s 2.3 s 0.9 s

Algoritmi 1 i 2 su identični osim što koriste različite strukture skupova. U ovom problemu taj izbor ima važan utjecaj na vrijeme izvršavanja jer je Algoritam 2 otprilike 4 do 5 puta brži od Algoritma 1. Međutim, najučinkovitiji algoritam je Algoritam 3 koji koristi sortiranje. On troši upola manje vremena u usporedbi s Algoritmom 2. Zanimljivo je da je vremenska složenost i Algoritma 1 i Algoritma 3 jednaka $O(n \log n)$, ali unatoč tome, Algoritam 3 je deset puta brži.

To se može objasniti činjenicom da je sortiranje jednostavan postupak koji se provodi samo jednom na početku Algoritma 3, dok ostatak algoritma radi u linearnom vremenu. S druge strane, Algoritam 1 održava kompleksno balansirano binarno stablo tijekom cijelog izvođenja algoritma.
